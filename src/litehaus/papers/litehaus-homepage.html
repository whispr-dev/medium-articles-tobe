Global Beacon Network

What is Litehaus?

Litehaus is a distributed network of lighthouse beacon servers deployed across four global regions. Each node broadcasts real-time performance data via WebSocket, allowing you to monitor network latency, JSON parsing speeds, and system health from anywhere in the world.

Built with high-performance C++ backends using SimdJSON for blazing-fast JSON processing, Litehaus demonstrates real-world distributed systems monitoring with sub-microsecond parsing times.

How It Works

Each Litehaus node runs a beacon service that continuously broadcasts JSON packets containing performance metrics. Connect via WebSocket to receive a live stream of beacon data including:

  • JSON parse time (microseconds)
  • Processing throughput (MB/s)
  • System uptime
  • Total beacons broadcast
  • Connected listeners count
  • CPU optimization level

Connecting to Litehaus

WebSocket Endpoints

Connect to any lighthouse beacon via WebSocket:

wss://nyc.litehaus.online/ws
wss://lon.litehaus.online/ws
wss://syd.litehaus.online/ws
wss://sgp.litehaus.online/ws

JavaScript Example

const ws = new WebSocket('wss://nyc.litehaus.online/ws');

ws.onopen = () => {
    console.log('Connected to NYC lighthouse');
};

ws.onmessage = (event) => {
    const beacon = JSON.parse(event.data);
    console.log('Beacon:', beacon);
};

ws.onclose = () => {
    console.log('Disconnected');
};

Command Line (wscat)

npm install -g wscat
wscat -c wss://nyc.litehaus.online/ws

Fastping Integration

Litehaus works alongside fastping.it.com for HTTP-based latency testing. While Litehaus provides continuous WebSocket beacon streams, Fastping offers on-demand ping/pong responses for measuring round-trip latency. 

curl -X POST https://fastping.it.com/ping \
    -H "Content-Type: application/json" \
    -d '{"timestamp": 1735012345678}'

Fastping returns the timestamp you sent along with server-side timing data, letting you calculate precise round-trip latencies.
